# OptimalSnow

Computes the optimal placement of **k snow deposit sites** for a city, given a total snow volume **V** and a set of candidate deposit locations. Each road segment is assigned to its nearest open deposit, subject to capacity constraints.

The problem is formulated as a **capacitated p-median**, solved with [OR-Tools](https://developers.google.com/optimization/) (CP-SAT or CBC).

---

## How it works

```
User inputs: total snow volume V, number of depots k, solver
                        ↓
     Compute snow demand per road: proportional to road area
                        ↓
     Cluster roads into spatial demand zones (KMeans)
                        ↓
     Solve capacitated p-median on cluster representatives
                        ↓
     Map cluster assignments back to individual streets
                        ↓
     Output: selected deposit sites + street assignment (GeoJSON)
```

### Problem formulation

- **Decision variables**: which k deposits to open; which deposit each road cluster is assigned to
- **Objective**: minimize total weighted travel distance (snow volume × distance to assigned deposit)
- **Constraints**: each cluster assigned to exactly one open deposit; deposit capacity not exceeded

## Project structure

```
OptimalSnow/
├── prepare_data.py          # One-time preprocessing: distance matrix, road fractions
├── clustering.py            # KMeans street clustering + cluster-level aggregation
├── main.py                  # Main pipeline (set user inputs here)
├── solvers/
│   ├── __init__.py          # Unified solver interface
│   ├── cp_sat.py            # OR-Tools CP-SAT solver

```

## Setup

**Python 3.11** recommended. Install dependencies:

```bash
pip install geopandas pandas numpy osmnx networkx scikit-learn pyarrow
pip install "ortools==9.11.*" "protobuf>=5.26,<6.0"
```

> **Note for macOS (Apple Silicon):** ortools 9.14+ has a protobuf compatibility issue. Use ortools 9.11 with protobuf 5.x as shown above.

## Usage

### Step 1 — Preprocess data (run once)

```bash
mkdir -p data_prepared
python prepare_data.py
```

This downloads the OSM road network for Tallinn, computes the full road-to-deposit distance matrix, and saves everything to `data_prepared/`. The distance matrix step can take several minutes.

**Input data required** (paths configured at the top of `prepare_data.py`):
- `tallinn_roads.geojson` — road segments with an `area_sqm` column
- `snow_capacity_municipality.geojson` — municipal deposit candidates with `snow_capacity_m3`
- `snow_capacity_government.geojson` — government deposit candidates with `snow_capacity_m3`

### Step 2 — Run the solver

Edit the user inputs at the top of `main.py`:

```python
total_snow_volume = 1_000_000  # m³ of snow to dispose
p = 5                          # number of deposits to open
solver_method = "cpsat"        # "cpsat" or "cbc"
```

Then run:

```bash
python main.py
```

### Outputs

Two GeoJSON files are written to the project root:

| File | Contents |
|------|----------|
| `results_deposits.geojson` | Selected deposit sites with all original attributes + `deposit_index` |
| `results_street_assignment.geojson` | All road segments with `assigned_deposit_index` and `cluster_id` |

`assigned_deposit_index` in the street file matches `deposit_index` in the deposits file.

## Solver options

| Method | Flag | Notes |
|--------|------|-------|
| CP-SAT | `"cpsat"` | Recommended. Parallel search, handles large instances well. Use `K=100` allowed depots per cluster. |
| CBC (MILP) | `"cbc"` | Exact MILP solver. Slower on large instances. Use `K=20` allowed depots per cluster. |

To switch to CBC, update `main.py`:
```python
solver_method = "cbc"
allowed_indices = build_allowed_assignments_cluster(cluster_dist_matrix, K=20)
```

---

## API integration

`main.py` is designed to be wrapped by an API. The three user-controlled variables (`total_snow_volume`, `p`, `solver_method`) at the top of the file are the only inputs that vary per job. The preprocessed data in `data_prepared/` is shared across all jobs.
